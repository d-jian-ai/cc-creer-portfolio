## 1. 项目定位

个人作品集网站。展示过去的设计、网页、视频作品，同时承载个人介绍与持续更新的文章。

**核心目标**：让访客在打开的头几秒就感受到作者的审美与技术能力，然后顺畅地看完作品。

**设计基调**：沉浸式体验 —— 全屏视觉、滚动驱动叙事、有质感的过渡动画。参考 igloo.inc 的完成度，但不照搬其重度 3D 方案。

**关键平衡**：动效服务于内容，不能盖过作品本身。任何一处动效如果让访客更慢看到作品，就砍掉。

---

## 2. 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 框架 | **Next.js 15**（App Router, TypeScript） | 多页面路由、图片优化、与 Vercel 原生集成 |
| 样式 | **Tailwind CSS** | 快速迭代，不用在文件间跳 |
| 动效 | **GSAP + ScrollTrigger** | 滚动叙事的行业标准 |
| 平滑滚动 | **Lenis** | 惯性滚动，沉浸感的基础 |
| 3D/着色器 | **Three.js**（按需，第三阶段再引入） | 首页视觉焦点 |
| 内容管理 | **MDX**（博客与作品详情） | 无需外部服务，内容随代码走 |
| 部署 | **Vercel** | 推送即部署 |

> Three.js 不在第一阶段引入。先把结构和内容跑通，视觉重头戏放后面。

## 3. 页面架构

```
/                     首页 — 沉浸式入口
/work                 作品集列表（可按类型筛选）
/work/[slug]          作品详情
/about                关于我
/writing              文章列表
/writing/[slug]       文章详情
/contact              联系
```

### 3.1 首页 `/`

分为若干全屏「幕」，随滚动推进：

1. **Hero** — 姓名 + 一句话定位。加载完成后有进场序列（文字逐字浮现 / 遮罩揭开）
2. **精选作品** — 3–4 件代表作，大图 + 悬停时的动态反馈
3. **能力简述** — 我做什么，三到四条，不堆砌技能标签
4. **最近文章** — 2–3 篇
5. **页脚 / 联系** — 邮箱与社交链接

### 3.2 作品集 `/work`

- 顶部按类型筛选：**全部 / 视觉设计 / 网页产品 / 视频动态**
- 网格布局，非等高（masonry 或不规则网格），避免呆板
- 悬停：视觉类显示更多画面，视频类自动播放静音预览
- 筛选切换时用 FLIP 动画重排，不要直接闪现

### 3.3 作品详情 `/work/[slug]`

内容块可自由组合（用 MDX 实现）：

- 全宽图 / 图组 / 视频嵌入
- 项目背景、我的角色、过程、结果
- 上一个 / 下一个作品导航

### 3.4 关于 `/about`

个人介绍、经历时间线、技能、照片。文字为主，动效克制。

### 3.5 文章 `/writing`

MDX 驱动。列表按时间倒序，显示标题、日期、摘要、阅读时长。

### 3.6 联系 `/contact`

邮箱直链 + 社交链接。表单可选（需要时用 Formspree 或 Vercel 的表单方案，不自建后端）。

## 4. 作品数据结构

作品类型混合，需要统一的元数据结构。每件作品是一个 MDX 文件，放在 `content/work/`：

```yaml
---
title: 项目名称
slug: project-slug
type: visual | web | video        # 决定卡片渲染方式
year: 2025
role: 设计 / 开发 / 导演
client: 客户或自主项目
summary: 一句话描述，出现在卡片和列表
cover: /images/work/xxx-cover.jpg
coverVideo: /videos/xxx-preview.mp4   # 可选，视频类作品的悬停预览
gallery:
  - /images/work/xxx-01.jpg
  - /images/work/xxx-02.jpg
liveUrl: https://...                  # 可选，网页类作品
featured: true                        # 是否出现在首页
order: 1
---

正文用 MDX 写，可以插入自定义组件。
```

**三种类型的卡片渲染差异**：

- `visual` — 静态封面，悬停时轮播 gallery 前几张
- `web` — 封面 + 悬停时显示访问链接，详情页嵌入实时预览 iframe（可选）
- `video` — 封面 + 悬停自动播放静音短预览，详情页嵌入完整视频

## 5. 交互与动效规划

### 必做（第二阶段）

- **平滑滚动**（Lenis）— 整站的基础手感
- **页面转场** — 路由切换时的遮罩过渡，避免白屏闪烁
- **进场动画** — 元素进入视口时的浮现，用 ScrollTrigger 批量处理
- **自定义光标** — 悬停可交互元素时变形/放大
- **加载序列** — 首屏资源预载 + 进度指示，加载完成后再播放进场动画

### 加分项（第三阶段，视精力）

- **首页着色器背景** — Three.js 全屏 shader，流动噪声 + 鼠标交互
- **文字撕裂/位移效果** — hero 标题的鼠标跟随扭曲
- **滚动驱动的镜头运动** — 3D 场景随滚动推进

### 明确不做

- 满屏粒子特效（性能差、审美过时）
- 强制的开场动画超过 2 秒
- 禁用滚动条或劫持滚动方向
- 背景音乐自动播放

---

## 6. 性能与降级

沉浸式体验的最大风险是加载慢和移动端卡顿。硬性要求：

- **首屏可见时间 < 2.5 秒**（4G 网络）。加载动画不能成为借口
- **移动端自动降级**：关闭 3D 场景与自定义光标，保留基础过渡
- **尊重 `prefers-reduced-motion`**：用户系统开了减弱动效，就只保留淡入淡出
- **图片全部走 Next.js `<Image>`**，自动生成 WebP 与多尺寸
- **视频预览控制在 3 秒内、2MB 以内**，`preload="none"`，进入视口才加载
- Lighthouse 性能分目标 **≥ 85**（有重动效的站，90+ 不现实）

## 8. 开发顺序

**每个阶段结束时项目都应该能跑起来、能看到效果。不要跳步。**

### 阶段 1 — 骨架（先跑通）

1. `create-next-app` 初始化（TypeScript + Tailwind + App Router）
2. 建 GitHub 仓库，首次提交
3. 接入 Vercel，确认推送后自动部署
4. 搭出所有页面的空壳 + 导航，能互相跳转
5. 定下字体、配色、间距体系

**验收**：线上能访问，所有页面能点开，样式统一。

### 阶段 2 — 内容系统

1. 配置 MDX 解析
2. 写 2–3 篇示例作品，跑通列表 → 详情的链路
3. 实现三种类型的卡片渲染
4. 实现筛选功能
5. 关于页、文章系统

**验收**：把真实作品填进去，站已经可用了。

### 阶段 3 — 动效层

1. Lenis 平滑滚动
2. ScrollTrigger 进场动画
3. 页面转场
4. 自定义光标
5. 加载序列

**验收**：手感到位，移动端不卡。

### 阶段 4 — 视觉重头戏（可选）

1. 首页 Three.js 着色器背景
2. Hero 文字交互效果
3. 性能调优与降级验证

### 阶段 5 — 上线打磨

- SEO 元信息、Open Graph 分享图
- favicon、站点地图
- 无障碍检查（键盘导航、对比度、alt 文本）
- 绑定自定义域名
- Lighthouse 跑分
