## 1. 项目定位

个人作品集网站。展示过去的**设计、网页、视频**作品，承载个人介绍与持续更新的思考文章。

**核心目标**：访客打开的头几秒就被氛围抓住，然后顺畅看完作品。

**设计基调**：沉浸式、自然、雾感、冷绿、安静、有呼吸感。参考 igloo.inc 的完成度。整站气质由「森林 + 天气」统领。

**关键平衡**：动效服务内容。任何动效如果让访客更慢看到作品，就砍掉。

---

## 2. 三大板块

用户构思的结构（注意：与传统多页导航不同，这是三个"世界"）：

```
加载页（森林 · 按天气变化 · 无文字）
   └─ 点击进入 ─┐
                ▼
   ┌── 首页 ──── 作品 ──── 空间 ──┐
   │  天气光晕背景   作品展示     可玩交互 │
   │  鼠标微交互     点击进详情   小游戏   │
   │  作者介绍       二级页       思考/练习 │
   └──────────────────────────────┘
```

- **首页**：渐变光晕背景（跟随鼠标微交互）+ 作者介绍。下滑后续内容待定。
- **作品**：点击不同作品 → 跳转对应二级页（作品展示 + 文字介绍）。列表动态形式待定。
- **空间**：沉浸式交互小游戏（氛围互动型，见 5.3）+ 作者的话、思考、练习记录。

## 3. 加载页（已明确，第一优先级）

这是整站的第一印象，也是最花心思的部分。**决策已定：**

| 项 | 决定 |
|---|---|
| 视觉 | **照片写实森林**（用户提供实拍图），叠加代码生成的动态雾气、光斑、天气粒子 |
| 天气 | **自动识别访客所在地天气**，森林随之切换氛围（雾/晴/雨/雪/夜） |
| 文字 | **完全无文字**。不出现名字、城市、标签。只有画面 + loading |
| loading | 精致的加载动效。候选：①细圆环+轨道光点 ②**森林从模糊/黑暗逐渐显影**（推荐，loading 与内容合一）。最终样式开发时再定 |
| 进入方式 | 加载完成后中央浮现"呼吸光点"，点击进入。转场：淡出 + 轻微缩放 |
| 视差 | 鼠标移动带动森林轻微缩放位移 + 光晕跟随 |

### 天气自动识别 — 技术方案

```
访客打开 → 按 IP 定位（城市级）→ 调天气 API → 森林切到对应氛围
```

- **API 用 Open-Meteo**（open-meteo.com）：免费、无需注册、无需 key、国内可访问。IP 定位可配合 ipapi.co 或浏览器 Geolocation（需用户授权，作氛围用建议直接用 IP，不弹权限）
- **必须有兜底**：API 失败 / 定位失败 / 超时 → 回落到默认森林（晨雾）。这是最容易翻车的地方，务必先写好降级
- 识别到的天气**存下来（如 sessionStorage）全站复用**，进入主站后氛围延续

### 已验证的原型

加载页原型已在规划会话中做过（矢量版 + 无文字照片版），实现了：分层视差、五种天气粒子（雨丝/雪花/萤火虫/光斑/雾）、鼠标光晕、圆环 loading、呼吸光点进入、暗角聚焦。**Code 开发时可要求参考这套交互逻辑重建为 React + 真实照片版。**

## 4. 进入后的氛围延续

加载页测到的天气状态带进主站：首页光晕、粒子颜色都跟当前天气走。晴天暖光斑、雨天冷调飘雨、雪天静谧、夜晚萤火。整站气质统一。

## 5. 页面与交互规划

### 5.1 首页 `/`

- 天气驱动的渐变光晕背景，鼠标微交互
- 作者介绍（第一屏）
- 下滑内容**待定**——建议先定内容再配动效
- 精选作品入口、最近文章入口

### 5.2 作品

- 列表：按类型筛选（全部 / 视觉设计 / 网页产品 / 视频动态）
- 三种类型卡片渲染不同：
  - `visual` 静态封面，悬停轮播画面
  - `web` 封面 + 悬停显示访问链接，详情页可嵌实时预览
  - `video` 封面 + 悬停自动播静音短预览，详情页嵌完整视频
- 二级页 `/work/[slug]`：作品展示（大图/图组/视频）+ 文字介绍（背景、角色、过程、结果）+ 上/下一个导航
- 列表动态形式**待定**

### 5.3 空间

- **沉浸式交互小游戏**，氛围互动型，与森林气质一脉相承
- 例：鼠标划过让植物生长 / 萤火虫聚集 / 拨开雾气。无输赢，耐看好玩
- 先做能玩的占位版，后续再深化
- 同板块放：作者的话、思考、练习记录（可用 MDX 文章形式）

## 7. 手机适配（必做）

**沉浸式站在手机上最容易翻车，从一开始就要考虑，不是最后补。**

- **响应式布局**：三大板块在手机上都要能正常浏览
- **性能降级**：手机端自动关闭/简化重动效——3D 场景降级或关闭、自定义光标关闭（触屏无光标）、天气粒子减量。保留核心氛围（森林图 + 雾 + loading）
- **交互适配**：鼠标视差 → 改用陀螺仪或轻触反馈；悬停效果 → 改为点击
- **"空间"小游戏**要有触屏版交互
- 尊重 `prefers-reduced-motion`
- 断点建议：手机 < 768px，平板 768–1024px，桌面 > 1024px

---

## 8. 性能红线

沉浸式的最大风险是慢和卡。硬要求：

- 首屏可见 **< 2.5 秒**（4G）。作品图多了尤其要注意，这也是加载页存在的意义
- 图片全走 Next.js `<Image>`，自动 WebP + 多尺寸
- 森林大图要压缩、按天气懒加载（只加载当前天气那张）
- 视频预览 < 3 秒 / < 2MB，`preload="none"`，进视口才加载
- Lighthouse 性能分 ≥ 85（重动效站 90+ 不现实）

## 9. 多语言切换（必做）

- 至少支持**中文 / 英文**切换（其他语言按需加）
- 用 **next-intl**（App Router 推荐）管理文案
- 所有界面文字、作品介绍、文章都要能切换；纯视觉内容（森林、动效）不受影响
- 语言选择器放导航里，记住用户选择
- 作品/文章的 MDX 建议按语言分文件或在 frontmatter 里分字段

## 10. 需要开发者提供的素材（重要 · 请逐项准备）

**AI 生成不了这些，需要你制作/提供。缺的先用占位，不要编造。**

### 🌲 加载页森林图（第一优先级）

- **理想**：几张**不同天气**的森林实拍图，一一对应：
  - 雾天（晨雾，也作默认兜底）
  - 晴天（有阳光穿透感）
  - 雨天（阴郁潮湿）
  - 雪天（可选）
  - 夜晚（可选）
- **最低**：至少一张高质量雾林图（像你之前发的那张），其余天气我靠调色模拟
- 要求：横构图、分辨率够大（长边 ≥ 2400px）、构图中心留白（放 loading）、**版权是你的或可商用**
- 格式：JPG/PNG，我会帮忙压缩

### 🖼 作品素材（第二优先级，作品集的血肉）

每件作品需要：

- 封面图（统一比例最好，如 16:10 或 4:3）
- 详情页用的图组 / 截图
- 视频类：预览短片（≤3s 静音）+ 完整视频（或 B站/YouTube 链接）
- 网页类：可访问的线上链接

### ✍️ 文字内容

- **作者介绍**（首页第一屏，几句话，可给草稿我润色）
- 每件作品：标题、类型、年份、你的角色、一段介绍
- "空间"板块：你的话 / 思考 / 练习记录（几段即可起步）
- 联系方式：邮箱、社交平台链接

### 🎨 品牌基调

- 主色倾向（默认跟森林走：冷绿 + 雾白 + 暖光点缀）
- 字体偏好（中英文各一，没有我推荐）
- 头像 / 个人照片（关于页用）

## 11. 作品数据结构（MDX）

放 `content/work/`，每件一个文件：

```yaml
---
title: 项目名称
slug: project-slug
type: visual | web | video
year: 2025
role: 设计 / 开发 / 导演
summary: 一句话描述（卡片和列表用）
cover: /images/work/xxx-cover.jpg
coverVideo: /videos/xxx-preview.mp4   # 可选，video 类悬停预览
gallery: [/images/work/xxx-01.jpg, /images/work/xxx-02.jpg]
liveUrl: https://...                  # 可选，web 类
featured: true                        # 首页是否精选
order: 1
# 多语言：可加 title_en / summary_en 等，或按语言分文件
---
正文 MDX，可插入自定义组件。
```

## 12. 开源资源（优先用，别造轮子）

**动效实现请先到这些地方找成熟方案参考：**

- **CodePen**（codepen.io）— 搜 "fog", "particles", "shader background", "loading animation", "forest", "parallax"，大量可直接改用的实现
- **GitHub 开源库**：
  - `pmndrs/drei` + `@react-three/fiber` — React 里用 Three.js 的标准方式（做 3D 森林/着色器）
  - `studio-freight/lenis` — 平滑滚动（已在技术栈）
  - `greensock/GSAP` — 动效（已在技术栈）
  - `pmndrs/react-spring` — 物理动画备选
  - `tsparticles` — 现成粒子系统（雨/雪/萤火虫可直接配）
- **着色器**：ShaderToy（shadertoy.com）找雾/光/水面 shader，移植到 Three.js
- **免费森林素材备选**（如果自制图不够）：Unsplash、Pexels（注意授权，商用需确认）
- **天气 API**：Open-Meteo（免费无 key）；IP 定位 ipapi.co

**原则**：先找现成的改，改不动再自己写。用之前确认许可证（MIT / 可商用）。

- 加载页放任何文字
